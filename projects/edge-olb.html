Language Python 3.8+
Project Type Capstone Project
Status Completed

Project Overview

This project provides a complete digital twin to model, simulate, and analyze the performance of Optimised Load Balancing (OLB) strategies in a fog computing environment for smart healthcare systems.

Implemented within the YAFS (Yet Another Fog Simulator) framework, the simulation creates a realistic IoT healthcare environment with geospatial awareness and evaluates how effectively the OLB algorithm distributes computational workloads across fog nodes.

Implementation Phases

Phase 1: Digital Twin Environment Construction

Geospatial Environment: Created 2D coordinate system (3000x2000 units)

IoT Layer (Tier 1): 10 sensor devices with OLB parameters:

Sensor Parameters
coordinates (x, y) tuple
transmissionPower P(x) Watts

flowTrafficSize l(x) megabits
averageFlowSize ν(x) MI
Fog Node Parameters
coordinates (x, y) static
processingPower Cj MIPS
bandwidth BWj MHz
carrierFrequency GHz
noisePower σ² Watts
Phase 2: Application Architecture Definition

YAFS Application Modules:

  • Client_Module: SOURCE type, handles sensor data ingestion
  • Processing_Module: MODULE type, performs data processing
  • Storage_Module: SINK type, stores processed results

Data Flow Topology: Sensor → Client_Module → Processing_Module → Storage_Module

Phase 3: OLB Algorithm Implementation
Communication Latency

Measures transmission time from sensor to fog node:

L_m(j) = TLj / (1 - TLj)
TLj = Σ eaj(x)
eaj(x) = (fl(x) * l(x)) / cj(x)
cj(x) = BWj * (1 + SNR(x))
SNR(x) = (P(x) * g(x)) / σ²
g(x) = 10 * log10(λ²/(4πd)²)
Computing Latency

Measures processing time at fog node:

L_p(j) = CLj / (1 - CLj)
CLj = Σ ebj(x)
ebj(x) = (fl(x) * ν(x)) / Cj

Final Selection: Fog node j with minimum L_total = L_m(j) + L_p(j)
