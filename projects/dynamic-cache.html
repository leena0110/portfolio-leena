Dynamic Cache System

High-performance caching system implementing LRU and LFU eviction policies with Doubly Linked Lists and HashMaps for optimal O(1) operations

Category Data Structures & Algorithms
Language Python
Timeline 2 Weeks

Project Overview

Caching is a fundamental technique used to improve the performance of software systems by storing frequently accessed data in a high-speed memory layer. This project implements a dynamic cache system with multiple eviction policies and a menu-driven interface for real-time interaction.

The system demonstrates efficient cache operations with O(1) time complexity for both read and write operations, achieved through intelligent data structure combinations.

System Architecture

The cache system combines three key data structures for optimal performance:

Doubly Linked List (DLL)

Maintains order of cache access, allowing O(1) insertion and deletion from both ends (head and tail). Efficiently tracks the most and least recently used items, essential for LRU eviction.

HashMap (Dictionary)

Enables O(1) average-time lookups of cache entries by key. Ensures fast access to nodes, facilitating quick operations in the cache.

Integrated Design

The DLL and HashMap work together to ensure efficient cache operations, with the DLL tracking the access order and the HashMap providing fast lookups.

Cache Write (Write-Through)

The write-through caching strategy ensures data consistency between cache and memory:

1

Client Write

When a client sends a write request to update or insert data, the cache layer intercepts the request before it reaches the underlying data store.

2

Cache Update

The cache layer updates the cached data to reflect the changes made by the client's write request.

3

Downstream Update

The cache layer then forwards the write request to the underlying data store, ensuring that the data is persistently updated in the primary data source.

Cache Access (Read-Through)

1

Client Request

When a client sends a request for data, the cache layer is the first point of access via the menu-driven process before it reaches the underlying data store.

2

Cache Lookup

The cache layer checks if the requested data is available in the cache. If found, it's immediately returned to the client with fast response time.

3

Cache Miss Handling

If data is not found in cache, it immediately notifies the user about the status and continues running until the user chooses to exit.

Cache Eviction Policies

1

LRU Policy

The LRU policy removes the least recently used item from the cache when capacity is reached. Implemented efficiently using doubly-linked list for constant-time operations.

2

Access Tracking

The cache maintains timestamps for each cached item. When eviction is required, the item with the oldest timestamp is removed.

3

Efficient Implementation

Combined with a data structure like a doubly-linked list, allowing for constant-time insertions, removals, and updates of access timestamps.

Performance Metrics Calculator

Hit Rate

Tracks the cache hit rate - ratio of successful cache lookups to total requests. High hit rate indicates efficient cache configuration.

Miss Rate

Monitors cache miss rate - ratio of cache misses to total requests. Low miss rate suggests effective serving of frequently accessed data.

Eviction Rates

Tracks cache eviction rates, providing visibility into effectiveness of chosen eviction policy and overall cache capacity management.

Time Complexity Analysis

Module Worst Time Complexity Justification
Cache Initialization O(1) Constant time for pointer manipulations and hash map initialization
Cache Access (get) O(1) HashMap provides constant-time lookups
Cache Insertion (put) O(1) Amortized Involves constant time insertion and potential eviction
LRU Eviction O(1) Doubly Linked List allows constant-time tail removal
Cache Display O(n) Traverses entire DLL; linear time relative to cache entries

Menu-Driven Interface

The system features an interactive console interface that allows users to:

  • Write to Cache: Insert or update key-value pairs with automatic eviction when capacity is reached
  • Access Cache: Retrieve values with real-time hit/miss tracking
  • Display Cache State: View current cache contents and performance metrics
  • Flush Cache to Memory: Write all cache contents to persistent storage
  • Real-time Metrics: Monitor hit ratios, miss ratios, and eviction rates

Technical Implementation

Core Classes:

  • DynamicCache: Main cache class integrating DLL and HashMap
  • CachePerformanceMetrics: Tracks and calculates performance statistics
  • CacheVisualizer: Provides visualization of cache state and metrics

Key Features:

  • Configurable cache capacity and eviction policy (LRU/LFU)
  • Write-through caching for data consistency
  • Real-time performance monitoring
  • Interactive user interface
  • Memory persistence options
